═══════════════════════════════════════════════════════════════════════════════
                         CREATING NEW MIGRATIONS IN FUTURE
═══════════════════════════════════════════════════════════════════════════════

When you need to make database schema changes:

1. Modify TypeScript Entity
   Edit: backend/src/modules/[feature]/entities/[entity].entity.ts
   Add/modify @Column() decorators

2. Build Project
   npm run build

3. Generate Migration (Auto-detects changes)
   npm run migration:generate -- -n DescriptiveName
   
   Example:
   npm run migration:generate -- -n AddPriceColumnToInventoryItem

4. Review Generated Migration
   Check file: backend/src/migrations/[timestamp]-DescriptiveName.ts
   Ensure up() and down() methods are correct

5. Execute Migration
   npm run migration:run

6. Commit to Git
   git add backend/src/migrations/[timestamp]-DescriptiveName.ts
   git commit -m "database: add price column to inventory items"
